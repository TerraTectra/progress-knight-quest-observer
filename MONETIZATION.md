# Monetization Plan

This project should stay playable and fair. Monetization must not sell direct progression, exclusive power, or mandatory convenience. The game is an idle/progression fork, so trust matters more than short-term conversion.

## Recommended model

1. Donations first
   - Add Boosty / Patreon / Ko-fi links outside the core gameplay loop.
   - Use wording like "support development" rather than "buy power".
   - Good for the first public beta because it does not require account systems.

2. Optional supporter cosmetics
   - Alternate themes that keep the original Progress Knight style.
   - Cosmetic title in the footer or settings page.
   - Optional Observer subject name packs.
   - Optional visual variants for Multiverse / Observer panels, only after preview approval.

3. Supporter-only preview builds
   - Early access to experimental balance branches.
   - No permanent exclusive mechanics in the public build.
   - Keep stable releases public.

4. Web ads only if needed
   - Avoid intrusive ads inside the game panel.
   - If ads are used, keep them outside the main play area and never use forced interstitials.
   - Disable ads for supporters.

## What not to sell

- Multiverse Points, Observer Points, Evil, Essence, Dark Matter, Hypercubes.
- x1000 speed as a paid feature.
- Faster offline progress as a paid-only feature.
- Exclusive upgrades or universes.
- Anything that makes free players feel like the balance is intentionally hostile.

## First implementation step

For v0.1.x, add a small "Support development" block in Settings with external links. This is a visual/UI change, so it must be previewed and approved before it goes live.

Suggested links to prepare:
- Boosty
- Patreon
- Ko-fi
- Discord

## Later implementation steps

- v0.2: supporter cosmetic themes and Observer subject name packs.
- v0.3: optional supporter preview branch.
- v0.4+: consider ads only if donations are not enough and only with a clean opt-out.
